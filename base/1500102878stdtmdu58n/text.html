Имя узла


Настраивается в файле /etc/hostname. Файл содержит одну строку с сетевым именем.



Статическая привязка IP адресов


Настраивается в файле /etc/hosts.


Формат файла: на каждой строке IP-адрес, дальше через табуляцию одно или несколько имен, которые привязываются к данному IP адресу.


Пример:


127.0.0.1 localhost

127.0.1.1 myserver.local.lan myserver



Файл перечисления сетей


Находится здаесь: /etc/networks. Обычное содержание такое:


default 0.0.0.0

loopback 127.0.0.0

link-local 169.254.0.0




Файл настройки DNS


Находится здесь: /etc/resolv.conf


Формат: строки из ключевых слов и значений. Разделители пробел или табуляция.


Пример:


domain local.lan

search local.lan

nameserver 77.88.8.8

nameserver 10.153.0.21


Ключевое слово nameserver может указываться более одного раза, обычно 2. С помощью него задается первый и второй DNS-сервер.


Ключевые слова domain и search используются только тогда, когда компьютер включается в домен (по LDAP или Microsoft AD). По-сути, это суффиксы, которые будут подставляться к именам, если разрешение короткого имени не сработает.



Файл настройки сетевых интерфейсов (самый главный файл)


Расположение: /etc/network/interfaces


Формат: ключевое слово, далее опции через пробел или табуляцию. Для комментирования используется символ #.


Пример для сервера, в котором находится две сетевые карточки:


# Интерфейс обратной петли

auto lo

iface lo inet loopback


# Интерфейс, смотрящий на интернет-шлюз

# Имеет маленькую общую подсетку 192.168.3.X со шлюзом

auto eth0

allow-hotplug eth0

iface eth0 inet static

address 192.168.3.2

netmask 255.255.255.0

# gateway 192.168.3.1 - Внимание, опция gateway может быть задана только для одного интерфейса!


# Интерфейс, смотрящий на локальную сеть

auto eth1

allow-hotplug eth1

iface eth1 inet static

address 10.163.0.31

netmask 255.255.0.0

gateway 10.163.0.1

post-up route add -net 172.16.155.0 netmask 255.255.255.0 gw 10.163.0.3
