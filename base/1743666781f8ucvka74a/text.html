В синтезаторе Yamaha E-473 используется микропроцессор SWX03S. Обозначение в докумнетации Yamaha - SCC105A, раrt. N: YK611A0. Внутренняя частота 135.4752MHz. Внешний тактовый генератор на 67.7376MHz. ПРедположительная архитектура - SuperH SH-2 RISC.


Что есть про этот микропроцессор в интернете?



1. Официальная документация


Руководство программиста SH-2 (англ. SuperH SH-2 Programming Manual):


— Говорят, что есть в архивах Renesas (Product Selector: RH850 Automotive MCUs, Product Selector: RX 32-Bit Performance / Efficiency MCUs). Там должны быть описаны регистры, инструкции и тайминги. По факту упоминаний архитектуры SuperH или SH-2 не найдено.


Технические справочники для конкретных чипов (например, HD64170) можно поискать на сайтах вроде DatasheetArchive.



2. Открытые проекты и эмуляторы


GNU Binutils (GAS, objdump):


— Поддерживает ассемблер и дизассемблирование SH-2.

— Исходный код в репозиториях GCC (GitHub).



Эмулятор MAME:


— Реализует SH-2 для аркадных автоматов (например, Sega Saturn).

— Код CPU-ядра: mame/src/devices/cpu/sh/sh2.cpp.



Проект QEMU:


— Частичная эмуляция SH-2 (менее точная, чем MAME).



3. Книги и учебные материалы


«See MIPS Run» (Dominic Sweetman):

— Сравнивает SH-2 с MIPS, объясняет RISC-принципы.



«Embedded Systems with ARM Cortex-M» (Joseph Yiu):


— Аналогии между SH-2 и ARM (полезно для понимания RISC-архитектур).



4. Онлайн-справочники


Wikibooks: SuperH:


— Краткий обзор инструкций.



CPUShack:


— Описание архитектуры.



Если нужно дизассемблировать код


Можно использовать утилиты:


— sh-elf-objdump (из пакета GNU Tools for SuperH). Взять его можно с https://github.com/andwn/sh-gcc-toolchain. Этот проект представляет собой просто один Makefile. При его сборке командой make, выкачиваются различные архивы с каких-то зеркал, на которых лежат исходники для Hitachi SuperH-2 (sh-elf). После сборки, файлы устанавливаются либо традиционной командой sudo make install, либо через checkinstall -D (для Debian-подобных После этого, по идее, становится доступна программа sh-elf-objdump. Где sh - это не сокращение от shell, а название архитектуры.


Например:


sh-elf-objdump -D -b binary -m sh2 firmware.bin



— Radare2/Ghidra с поддержкой SH-2. В Ghidra v. 11.3.1 точно есть поддержка архитектур SuperH и SuperH4. Что обозначает цифра 2 в строке "SH-2" - непонятно. То ли архитектуру, то ли количество ядер.
