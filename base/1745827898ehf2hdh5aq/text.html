В Astra Linux Special Edition 1.7 и Astra Linux Special Edition 4.7 изменен вариант поставки:



  • основной репозиторий (main) — соответствует сертифицированному ФСТЭК изделию (установочный диск);
  • базовый репозиторий (base) — содержит в себе основной репозиторий и средства разработки.



Отдельного диска, содержащего только средства разработки, нет.



* * *



Чтобы подключить базовый (base) репозиторий в системе с доступом к интернету, следует воспользоваться инструкцией из статьи Справочного центра Интернет-репозитории Astra Linux Special Edition x.7.


Чтобы подключить базовый репозиторий в системе без доступа к интернету, необходимо:



1.


Загрузить архив базового репозитория по ссылке, доступной в Личном кабинете в разделе "Дистрибутивы" при наличии действующей лицензии на продукт.



Внимание!

При наличии установленного оперативного обновления в системе необходимо дополнительно использовать оперативное обновление диска со средствами разработки с тем же порядковым номером!


Подробная информация об оперативных обновлениях доступна в Справочном центре.



2.


Если каталог /opt/repo/ не существует — создать его:



sudo mkdir -p /opt/repo/


В каталоге /opt/repo/ будут размещены файлы репозитория.


3.

Распаковать архив репозитория в каталог /opt/repo/:


sudo tar xvf <имя_архива> -C /opt/repo


4.

Создать в каталоге /etc/apt/sources.list.d/ файл base.list следующего содержания:


Для Astra Linux Special Edition 1.7:


Файл: /etc/apt/sources.list.d/base.list

deb file:///opt/repo/ 1.7_x86-64 contrib main non-free


Для Astra Linux Special Edition 4.7:

/etc/apt/sources.list.d/base.list

deb file:///opt/repo/ 4.7_arm contrib main non-free


5.

Обновить список доступных пакетов:


sudo apt update
